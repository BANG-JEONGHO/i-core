# NFR 요구사항 — unit-matching-core

## 성능 (Performance)

| 요구사항 | 기준 | 비고 |
|---|---|---|
| 문서 파싱 시간 | < 3초 (50MB 이하 파일) | 매칭 전체 5초 중 파싱에 할당 |
| 매칭 계산 시간 | < 2초 (108명 강사 풀) | 키워드+규칙 기반 1단계 |
| 메모리 사용량 | < 500MB (파싱 + 매칭 처리 시) | 단일 요청 기준 |

## 확장성 (Scalability)

| 요구사항 | 기준 | 비고 |
|---|---|---|
| 강사 풀 규모 | 현재 108명 → 최대 200명까지 | 선형 확장 가능 설계 |
| 파일 크기 | 최대 50MB | 메모리 효율적 파싱 필요 |
| 동시 매칭 요청 | 10건 이하 | 사내 도구 수준 |

## 보안 (Security) — Security Baseline 준수

| 규칙 | 적용 여부 | 적용 내용 |
|---|---|---|
| SECURITY-03 (로깅) | 적용 | 구조화된 로깅, 민감 정보 미포함 |
| SECURITY-05 (입력 검증) | 적용 | 파일 형식/크기 검증, 악성 파일 방어 |
| SECURITY-09 (에러 처리) | 적용 | 내부 정보 미노출, 제네릭 에러 메시지 |
| SECURITY-10 (공급망) | 적용 | 의존성 버전 고정 (requirements.txt) |
| SECURITY-15 (예외 처리) | 적용 | 모든 외부 호출에 명시적 에러 핸들링 |

## 신뢰성 (Reliability)

| 요구사항 | 기준 |
|---|---|
| 파싱 실패 시 | 명확한 에러 반환 (시스템 중단 없음) |
| 매칭 실패 시 | 빈 결과 반환 + 에러 로그 |
| 비정상 파일 처리 | 손상/보호 파일 감지 후 안전하게 거부 |

## 유지보수성 (Maintainability)

| 요구사항 | 기준 |
|---|---|
| 모듈 구조 | 전략 패턴으로 매칭 알고리즘 교체 가능 |
| 테스트 용이성 | 순수 함수 위주 설계, 외부 의존 최소화 |
| 문서화 | 공개 API에 docstring 필수 |
| 코드 구조 | 파서와 엔진 완전 분리 |

## 테스트 (Property-Based Testing — Partial 적용)

| 대상 | PBT 적용 |
|---|---|
| 점수 계산 함수 (순수 함수) | 적용 — 점수 범위 불변식 검증 |
| 키워드 매칭 함수 (순수 함수) | 적용 — 교환 법칙, 단조성 검증 |
| 파서 출력 형식 | 적용 — 출력 구조 불변식 검증 |
| 파일 I/O | 미적용 — 외부 의존 |
