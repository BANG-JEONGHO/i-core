# 비즈니스 규칙 — unit-frontend

## 인증 규칙

### BR-F01: 토큰 관리
- JWT 토큰은 localStorage에 저장
- 앱 시작 시 토큰 존재 여부 확인 → 없으면 /login 리다이렉트
- 401 응답 수신 시 토큰 삭제 + 로그인 페이지 이동

### BR-F02: 인증 UI
- 로그인/회원가입 페이지는 인증 불요
- 나머지 모든 페이지는 ProtectedRoute로 감싸기
- 로그아웃 시 토큰 삭제 + 상태 초기화

---

## 파일 업로드 규칙

### BR-F03: 강사 파일 업로드
- 허용 형식: .xlsx, .csv
- 최대 크기: 10MB
- 업로드 전 형식/크기 클라이언트 검증
- 업로드 중 프로그레스 표시
- 결과: 성공/실패 건수 표시

### BR-F04: 과업지시서 파일 업로드
- 허용 형식: .pdf, .hwp, .docx
- 최대 크기: 50MB
- 드래그앤드롭 지원
- 업로드 후 파싱 결과 자동 표시

---

## 매칭 결과 규칙

### BR-F05: 결과 표시
- 전체 강사를 점수 내림차순으로 목록 표시
- 점수 색상 코딩: >=70 녹색, 40~69 노란색, <40 빨간색
- 강사 클릭 시 매칭 상세 카드 팝업/사이드패널

### BR-F06: 매칭 카드
- 종합 점수 + 항목별 점수 바 차트
- 강점/약점 목록
- 매칭된 키워드 하이라이트
- 한줄 요약

---

## UI/UX 규칙

### BR-F07: 반응형
- 최소 너비: 1024px (데스크탑 사내 도구)
- 사이드바 축소 가능

### BR-F08: 한국어 인터페이스
- 모든 텍스트 한국어
- 에러 메시지 한국어
- 날짜 형식: YYYY.MM.DD

### BR-F09: 로딩/에러 상태
- API 호출 중: 로딩 스피너 표시
- API 에러: 토스트 알림으로 에러 메시지 표시
- 빈 상태: 안내 메시지 + 행동 유도 버튼
