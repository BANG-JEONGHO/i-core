# 요구사항 문서 — 강사 매칭 플랫폼

## Intent Analysis Summary

| 항목 | 내용 |
|---|---|
| **사용자 요청** | 교육기업의 강사 매칭 웹 플랫폼 개발 |
| **요청 유형** | New Project (신규 프로젝트) |
| **범위** | Multiple Components (프론트엔드 + 백엔드 + AI 매칭 엔진 + 문서 파서) |
| **복잡도** | Moderate-High (AI 매칭, 문서 파싱, 단계적 구현) |
| **프로젝트 유형** | Greenfield |

---

## 프로젝트 개요

교육기업에서 나라장터 과업지시서를 업로드하면, 회사 소속 강사(약 108명)의 이력서 데이터를 기반으로 최적의 강사를 매칭·추천해주는 내부 업무 웹 플랫폼.

---

## 기술 스택

| 계층 | 기술 |
|---|---|
| **프론트엔드** | React + TypeScript |
| **백엔드** | Python FastAPI |
| **AI/매칭** | 단계적 구현 (키워드+규칙 → 임베딩 → Knowledge Graph + Agent) |
| **데이터 소스** | Excel/CSV (강사 이력서 108명) |
| **배포** | 미정 (개발 후 결정) |

---

## 기능 요구사항 (Functional Requirements)

### FR-01: 사용자 인증
- 회사 계정 기반 로그인 시스템
- 세션 관리 및 로그아웃 기능
- 비밀번호 해싱 및 보안 저장
- 단일 역할 (별도 권한 구분 없음)

### FR-02: 강사 이력서 데이터 관리
- Excel/CSV 파일 업로드를 통한 강사 데이터 등록
- 강사 정보 조회 (목록, 상세)
- 강사 정보 수정/삭제
- 저장 데이터: 이름, 전문 분야, 교육 과목, 경력 연수, 자격증, 기타 이력 정보

### FR-03: 과업지시서 업로드 및 파싱
- 지원 형식: PDF, 한글(.hwp), Word 등 혼합 형식
- 과업지시서에서 핵심 정보 자동 추출:
  - **평가기준** (배점, 기술평가 항목 등)
  - **참여자격/신청자격** (요구 자격증, 경력 조건, 학력 등)
- 추출된 정보 미리보기 및 사용자 확인/수정 기능

### FR-04: 강사 매칭 엔진 (단계적 구현)

#### 1단계: 키워드 + 규칙 기반 매칭
- 과업지시서 참여자격과 강사 이력서 키워드 매칭
- 가중치 점수 시스템 (전문 분야 일치, 경력/자격증 매칭)
- 점수 기반 정렬

#### 2단계: AI 임베딩 기반 매칭
- OpenAI API 또는 유사 임베딩 모델 활용
- 의미 기반 유사도 계산
- 키워드 매칭과 임베딩 매칭 점수 결합

#### 3단계: Knowledge Graph + AI Agent
- 강사-기술-자격증-경력 관계를 Knowledge Graph로 모델링
- AI Agent를 통한 매칭 결과 검증 및 설명 생성
- 매칭 추천 근거 자동 생성

### FR-05: 매칭 결과 표시
- 점수 기반 목록으로 전체 강사를 정렬하여 표시 (직관적 UI)
- 각 강사 클릭 시 카드 형식으로 매칭 이유/근거 표시
- 매칭 점수 breakdown (어떤 항목에서 점수를 얻었는지)

### FR-06: 대시보드
- 최근 매칭 이력 조회
- 과업지시서 업로드 이력
- 강사 데이터 통계 (전문 분야별 분포 등)

---

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 매칭 결과 응답 시간: 1단계 기준 5초 이내
- 동시 접속 사용자: 회사 전체 직원 규모 (수십 명 수준)
- 파일 업로드 용량: 과업지시서 최대 100MB

### NFR-02: 보안 (Security Extension 적용)
- SECURITY-01 ~ SECURITY-15 규칙 준수
- 회사 계정 인증 (비밀번호 해싱, 세션 보안)
- 입력값 검증 및 SQL Injection 방지
- HTTPS 통신 강제
- HTTP Security Headers 적용
- Rate Limiting 적용
- 구조화된 로깅 (민감 정보 제외)

### NFR-03: 사용성
- 직관적 UI (기술 비전문가도 사용 가능)
- 한국어 인터페이스
- 반응형 웹 디자인

### NFR-04: 유지보수성
- 모듈화된 매칭 엔진 (단계별 교체/추가 용이)
- API 문서 자동 생성 (FastAPI Swagger)
- 코드 문서화

### NFR-05: 확장성
- 강사 데이터 규모 증가 대응 (현재 108명 → 향후 확장)
- 매칭 알고리즘 단계적 고도화를 위한 플러그인 구조

## 데이터 모델 (개요)

### 강사 (Instructor)
- ID, 이름, 전문 분야, 교육 과목(복수), 경력 연수, 자격증(복수), 학력, 연락처, 기타 정보

### 과업지시서 (Task Order)
- ID, 파일명, 업로드일시, 원본파일경로, 추출된 평가기준, 추출된 참여자격

### 매칭 결과 (Matching Result)
- ID, 과업지시서 ID, 강사 ID, 매칭 점수, 매칭 근거, 생성일시

---

## 제약사항

- 과업지시서 형식이 다양하여 파싱 정확도에 한계가 있을 수 있음
- 한글(.hwp) 파싱을 위한 별도 라이브러리 필요
- 1단계에서는 AI API 의존 없이 독립 동작 가능해야 함
- 배포 환경 미정 — 로컬 개발 환경에서 완전 동작 필수
